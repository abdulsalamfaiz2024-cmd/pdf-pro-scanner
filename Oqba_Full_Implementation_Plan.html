SHARED · CRITICAL lib/services/page_thumbnail_service.dart NEW FILE

Renders PDF pages to images using pdfrx. Implements an in-memory LRU cache to avoid re-rendering the same page twice. All page selector UIs share this single service.

lib/services/page_thumbnail_service.dart
import 'dart:typed_data';
import 'package:pdfrx/pdfrx.dart';

class PageThumbnailService {
  // In-memory LRU cache: key = "pdfPath::pageIndex::width"
  final Map<String, Uint8List> _cache = {};
  static const int _maxCacheEntries = 60; // ~12 pages × 5 PDFs

  /// Renders a single PDF page to PNG bytes.
  /// [width] controls output resolution (default 200px wide, height auto).
  /// Returns null on failure — callers must show a placeholder.
  Future<Uint8List?> getThumbnail(
    String pdfPath,
    int pageIndex, {
    int width = 200,
  }) async {
    final cacheKey = '$pdfPath::$pageIndex::$width';
    if (_cache.containsKey(cacheKey)) return _cache[cacheKey];

    try {
      final doc = await PdfDocument.openFile(pdfPath);
      final page = doc.pages[pageIndex];
      final pageImage = await page.render(
        fullWidth: width.toDouble(),
        fullHeight: (width * page.height / page.width),
        backgroundColor: 0xFFFFFFFF,
      );
      final bytes = await pageImage.createImageIfNotAvailable()
          .then((img) => img.toByteData(format: ui.ImageByteFormat.png))
          .then((bd) => bd!.buffer.asUint8List());
      doc.dispose();

      // Evict oldest entry if over limit
      if (_cache.length >= _maxCacheEntries) {
        _cache.remove(_cache.keys.first);
      }
      _cache[cacheKey] = bytes;
      return bytes;
    } catch (_) {
      return null;
    }
  }

  /// Call when a PDF file changes (after split, delete, rotate etc.)
  void evictPdf(String pdfPath) {
    _cache.removeWhere((key, _) => key.startsWith('$pdfPath::'));
  }

  void clearAll() => _cache.clear();
}
Architecture Rule
Instantiate this as a singleton via a global final or an InheritedWidget at the MaterialApp root. Do NOT instantiate per-page — the cache is the value.

class CompressService {

  // Compression presets
  static const Map<String, _CompressionPreset> presets = {
    'low':    _CompressionPreset(label: 'Low',    jpegQuality: 85, syncLevel: PdfCompressionLevel.best),
    'medium': _CompressionPreset(label: 'Medium', jpegQuality: 65, syncLevel: PdfCompressionLevel.best),
    'high':   _CompressionPreset(label: 'High',   jpegQuality: 40, syncLevel: PdfCompressionLevel.best),
  };

  /// Compresses a PDF using a two-stage approach:
  ///
  /// STAGE 1 — Syncfusion native compression:
  ///   doc.compressionLevel = preset.syncLevel;
  ///   // This compresses streams, removes duplicate objects, optimizes xref
  ///
  /// STAGE 2 — Image resampling (the major size reduction):
  ///   For each page, extract embedded images, re-encode with flutter_image_compress
  ///   at preset.jpegQuality, then embed the smaller version back.
  ///
  /// NOTE: Stage 2 is the heavyweight operation. Run in compute() isolate.
  /// Input PDFs with no embedded images (text-only) will only benefit from Stage 1.
  ///
  /// Returns a CompressResult with: outputPath, originalSizeBytes, newSizeBytes
  Future<CompressResult> compressPdf(
    String pdfPath,
    String preset, // 'low', 'medium', 'high'
  ) async { ... }
}

class CompressResult {
  final String outputPath;
  final int originalSizeBytes;
  final int newSizeBytes;
  int get savedBytes => originalSizeBytes - newSizeBytes;
  double get reductionPercent => (savedBytes / originalSizeBytes) * 100;
}

09 Agent Execution Order

Optimal Build Sequence

Follow this order strictly. Each group unblocks the next. Parallel execution is possible within each group.

1
Dependencies → Build Verification
Add all 4 new packages to pubspec.yaml. Run flutter pub get. Run flutter build apk --debug. Fix any Gradle/Kotlin conflicts. Do not proceed until build is green.
2
Shared Infrastructure (Group — can parallel)
Build page_thumbnail_service.dart + page_selector_grid.dart + pro_gate.dart. Test PageThumbnailService in isolation with a known PDF before connecting to any UI.
3
Extend PdfService (all 5 organize functions)
Add split, extractPages, deletePages, rotatePages, reorderPages to pdf_service.dart. Write a quick Dart unit test for each using 2 pre-made test PDFs. All 5 must produce valid PDF output before moving on.
4
New Services (Group — can parallel)
Build convert_service.dart + extract_service.dart + compress_service.dart + security_service.dart (stub). Test each service function in isolation before connecting UI.
5
Organize UI Pages (Group — can parallel)
Build all 5 organize pages: split, extract_pages, reorder, delete, rotate. All depend on PageSelectorGrid (built in Step 2). Connect to their PdfService functions.
6
Convert UI Pages (Group — can parallel)
Build pdf_to_images, images_to_pdf, ocr pages. Wrap ocr_page in ProGate. Connect to ConvertService.
7
Extract & Optimize UI Pages (Group — can parallel)
Build extract_images, extract_text, compress pages. Wrap compress_page in ProGate. Connect to ExtractService and CompressService.
8
Security Stub Page
Build the polished security_page.dart Coming Soon screen. Takes 20 minutes. Polish it properly.
9
Update main.dart Routes + tools_page.dart Grid
Add all routes to the route table. Update tools_page to full categorized grid. This step is last because it references all page files — they must exist.
10
Full Test Matrix on Physical Device
Execute every row in Section 08's test matrix. Fix crashes and visual glitches only. No new features.
11
Release Build
flutter build apk --release --target-platform android-arm64. Install release APK (not debug) on demo device. Verify the complete demo loop one final time on the release build.
⚠
File Contract Reminder — Zero Tolerance
Every single output file from every service must land in the correct subdirectory:
AppDocuments/Oqba/documents/ — all output PDFs
AppDocuments/Oqba/extracted/ — all extracted images, text files, converted images
AppDocuments/Oqba/scans/ — raw scanner output (existing, do not change)
AppDocuments/Oqba/temp/ — intermediate processing files (always clean up after use)
Never write to the system Downloads folder or any path outside these directories. Never pass external content:// URIs to service functions.
